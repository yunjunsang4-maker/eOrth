# 빈 소셜탭 리디자인 (eOrth 공식 예시 콘텐츠) 설계

**작성일:** 2026-07-17

## 배경 · 목표

신규 사용자가 소셜탭을 처음 열면 이웃·기록이 없어 피드가 비고, 현재는 안내 문구 + 추천 친구 + **"친구 찾기" CTA**가 뜬다. 문제:
- (A) 첫인상이 허전·외롭고 "사람부터 찾아라"를 강요하는 느낌 → 앱 가치를 못 보여줌.
- (C) eOrth 정체성의 뿌리인 **"나만의 기록"**을 첫 화면이 안 보여주고 소셜부터 들이밈 → 순서가 뒤집힘.

**목표:** 빈 소셜탭을 "기록을 남기면 여기 쌓이고 이웃과 나눠진다"로 재구성하고, **eOrth 공식 예시 콘텐츠**(예시 기록 + 기능 소개 + 데모 스냅)를 네이티브처럼 보여줘 튜토리얼 없이 형식·기능을 체감하게 한다.

**비목표:** 서버·RLS 변경 없음(닫힌 피드 모델 유지). 팔로우 가능한 실제 공식 계정(@eorth DB 프로필)은 만들지 않음.

## 확정 결정 (Locked Decisions)

1. **번들 공식 콘텐츠** — 실 서버 글 아님. 클라이언트가 피드가 빌 때 주입. 작성자 표기 "eOrth" + **공식 배지**(정직 표식). RLS/서버 무변경.
2. **빈 상태 재구성(A)** — "친구찾기" 중심 → "기록을 남기면 여기서 이웃과 나눠져요" + 주 CTA "첫 기록 남기기"(지구본/기록작성). 친구찾기 보조.
3. **피드 예시 2장**(둘 다 피드 형식, 공식 배지):
   - (a) **예시 기록 카드** — 진짜 게시물처럼(번들 사진). 형식 힌트 한 줄. 탭 → 상세 + "나도 남기기". 좋아요·댓글·신고·차단·작성자 이동 **비활성**.
   - (b) **기능 소개 카드(자동 슬라이드)** — **지구본·통계·배지** 3슬라이드 순회. **탭 없음**(정보만). 3초 자동 전환 + 스와이프, 화면 밖 정지, 점 인디케이터.
4. **데모 스냅** — eOrth 제공 사진 1장. 내 스냅 0개일 때 스냅 링에 노출. 탭 → 예시 스토리.
5. **수명주기** — 예시 피드 2장은 `allVisible.length===0`(완전히 빈 피드)일 때만. 실 글/이웃 콘텐츠 생기면 사라짐. 데모 스냅은 내 스냅 0개일 때.

## 데이터 · 자산

- 예시 기록: `TravelRecord` 형태 상수 + 번들 이미지. **단일 플래그 `isExample: true`** 가 모든 것을 구동한다(공식 배지 표시·상호작용 비활성·프로필 이동 차단). 작성자 `user.handle='eorth'`. 실제 발행/서버 저장 안 함 — 순수 로컬 렌더용.
- 기능 슬라이드: `{ image, titleKey, descKey }[]`(지구본/통계/배지) 상수. 문구는 i18n(ko/en).
- 데모 스냅: `TravelRecord`(viewType='snap') 형태 + `isExample:true`, 번들 이미지 1장.
- 신규 상수 파일: `src/constants/exampleContent.ts`(예시 기록·기능 슬라이드·데모 스냅 정의), 이미지 에셋은 `assets/example/`.

## 렌더 통합 (SocialScreen)

현재 구조(참조): 빈 상태 `allVisible.length===0`(2599행) emptyWrap; 피드 컬럼 `columns`(2509)·masonry 렌더(2642, `_adSlot`→FeedAdCard/else DiaryCard); 스냅 링 `snapItems`(2437·2550); 하우스광고 `getHouseAd`/`FeedAdCard`.

- **빈 상태**: emptyWrap의 문구·CTA를 결정 2로 교체. 추천 친구·친구찾기는 하단 보조로 유지(축소).
- **예시 피드 2장**: `allVisible.length===0`일 때, 피드 masonry에 예시 기록 카드(a) + 기능 소개 카드(b)를 주입해 네이티브하게 배치. 예시 기록은 기존 `DiaryCard`로 렌더하되 `isExample` 가드로 상호작용 비활성·공식 배지 표시. 기능 카드는 신규 컴포넌트.
- **데모 스냅**: `snapItems`에 내 스냅이 없을 때 데모 스냅을 앞에 주입(공식 배지 링). 탭 → 기존 스냅 스토리 뷰어로 예시 표시.

- 생성: `src/components/social/FeatureShowcaseCard.tsx` — 자동 슬라이드 기능 소개 카드. 3초 timer + PanResponder 스와이프, `useAnimationsActive`로 화면 밖 정지(발열 방지, 기존 패턴), 점 인디케이터. 피드 카드와 같은 폭/모서리로 네이티브.
- 수정: `src/screens/SocialScreen.tsx` — 빈 상태 재구성, 예시 카드·데모 스냅 주입, `isExample` 상호작용 가드.
- 수정: `DiaryCard`(SocialScreen 내부 카드 렌더) — `isExample`면 좋아요/댓글/신고/차단/작성자 탭 비활성 + "eOrth 공식" 배지. PostDetail 탭은 record 폴백으로.
- 생성: `src/constants/exampleContent.ts` + `assets/example/*`
- 수정: `src/i18n/locales/ko.ts`·`en.ts` — 빈 상태 문구·형식 힌트·기능 슬라이드 문구·공식 배지.

## 정직성 · 상호작용 규칙

- 모든 예시 콘텐츠에 **"eOrth 공식" 배지** — 진짜 유저 글과 구분되게(신뢰 보호). 네이티브 90% / 표식 10% 균형.
- 예시 기록/스냅은 **읽기 전용** — 좋아요·댓글·신고·차단 비활성, 작성자 탭 시 프로필 이동 대신 무시(또는 "eOrth 공식" 안내). 서버 호출 일절 없음.
- 기능 소개 카드는 탭 없음(정보 전용).

## 엣지 케이스

- 사용자가 첫 기록을 남기면 `allVisible`이 채워져 예시 피드 2장 자동 소멸. 첫 스냅을 올리면 데모 스냅 소멸.
- 예시 카드가 좋아요 수 등 표시하더라도 정적 값(서버 반영 없음). `showCounts` 설정과 무관하게 상호작용은 비활성.
- 오프라인: 번들이라 정상 노출.
- i18n: 형식 힌트·기능 문구 ko/en 동일 키.

## 검증

- `npx tsc --noEmit`.
- FeatureShowcaseCard의 슬라이드 순환·정지 로직은 순수 부분(다음 인덱스 계산)만 `*.verify.ts`로, 애니는 실기기 확인.
- 수동 플로우: 신규(기록·이웃 0) 소셜탭 → 재구성된 빈 상태 문구·CTA, 예시 기록 카드·기능 슬라이드 카드·데모 스냅 노출·공식 배지·상호작용 비활성 확인 → 첫 기록 남기면 예시 소멸.

## 롤아웃 노트

- 순수 클라이언트(서버·SQL 무변경). 기존 사용자(피드에 콘텐츠 있음)는 예시 미노출 — 영향 없음.
- 콘텐츠(사진·문구)는 번들이라 갱신 시 앱 업데이트 필요. 후속으로 원격설정(하우스광고처럼) 승격 가능.
- 예시 이미지는 사용자(앱 소유자)가 직접 제공 — 스톡 느낌 안 나게 실제 사진.
