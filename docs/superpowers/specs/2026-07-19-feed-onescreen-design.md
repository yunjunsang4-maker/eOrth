# 피드 기록 한 화면 통합 + 사진별 글 설계

날짜: 2026-07-19
상태: 사용자 승인 완료 (브레인스토밍 세션에서 단계별 확정)

## 개요

피드 형식 기록 작성(NewRecordScreen)의 3단계 마법사(국가 → 사진 → 내용)를 **하나의
스크롤 화면**으로 통합하고, 글 구조를 **기록 전체 본문 1개 → 사진별 글**로 바꾼다.
보기 화면(PostDetail)도 사진 캐러셀 + 사진별 글로 함께 개편한다.

## 확정 결정 사항

| 항목 | 결정 |
|------|------|
| 화면 구조 | 단계 제거, 한 화면 스크롤 (A안) |
| 검증 | 저장 시 일괄 검증 + 하단 고정 저장 바(미충족 시 흐림, 눌러도 안내) |
| 섹션 순서 | 사진(+사진별 글) → 국가 선택 → 여행 정보 |
| 사진 표시 | 최대한 크게 — 화면 폭 페이저, 좌우 스와이프 |
| 글 구조 | **사진별 글이 전부** — 기록 전체 본문 폐지 (A안) |
| 보기 화면 | PostDetail도 캐러셀+사진별 글로 개편, 목록 미리보기는 대표 사진 글 (A안) |
| 하위 호환 | 옛 기록(단일 글)은 기존 렌더 유지, 마이그레이션 없음 |

## ① 작성 화면 (NewRecordScreen 리레이아웃)

위→아래 순서의 단일 ScrollView:

1. **큰 사진 페이저**
   - 화면 폭 전체 사진, 좌우 스와이프(다음 사진 일부 노출 + 도트 인디케이터 + n/N 표시)
   - 사진 추가(＋)·삭제·대표 지정은 페이저 안에서 처리 (기존 MediaPicker 재사용)
   - **페이저 바로 아래 글 입력칸** — 현재 보이는 사진의 글을 편집. 사진을 넘기면
     입력칸 내용이 그 사진의 글로 전환된다 ("이 사진의 글 (n/N)" 라벨)
2. **국가 선택** — 선택 전: 기존 검색·목록 UI. 선택 후: "국기 국가 · 지역 [변경]" 칩으로
   접힘. 바로 아래 여행 기억 서랍(MomentDrawer) 유지(매칭 근거인 국가·날짜가 이 부근)
3. **여행 정보** — 날짜(시작~종료) + 동행자 + 별점·날씨 등 기존 선택 항목

- 진행 바(1-2-3)·이전/다음 버튼 제거
- **하단 고정 저장 바**: 필수 충족 전 흐림(opacity). 흐린 상태에서 눌러도 반응 —
  첫 미충족 섹션으로 스크롤 + 섹션 강조(테두리·안내 문구)
- 편집 모드(editRecord)도 동일 레이아웃 사용
- 여행 카드 경유 진입(tripPeriod·selectedCountry 파라미터) 동작 유지

### 필수 조건 (저장 시 일괄 검증)
- 국가 1개 이상
- 사진 1장 이상 + 대표 사진 지정
- **사진별 글은 선택 사항** — 글 없는 사진 허용. 기존 "내용 필수" 규칙은 폐지
  (정책 변화, 사용자 확정)

## ② 데이터 구조

- `TravelRecord`에 `photoTexts?: string[]` 추가 — **medias와 index가 짝인 병렬 배열**.
  같은 사진을 두 번 추가하는 경우까지 안전하도록 uri 키 매핑이 아닌 병렬 배열로 하고,
  사진 추가·삭제·순서 변경은 반드시 medias와 photoTexts를 함께 조작하는
  단일 헬퍼를 거치게 해서 어긋남을 원천 차단한다
- **하위 호환 장치**: 저장 시 대표 사진의 글을 기존 `text`(본문) 필드에도 복사한다.
  피드 목록 미리보기·검색·서버 백업 등 기존 소비처는 수정 없이 동작
- 옛 기록(사진별 글 없음)은 그대로 둔다 — 데이터 마이그레이션 없음

## ③ 보기 화면

- **PostDetail**: 사진 캐러셀(작성 페이저와 같은 감각) + 현재 사진의 글 표시.
  사진을 넘기면 글도 전환. 사진별 글이 없는 사진은 글 영역 생략
- **피드 목록 카드**: 대표 사진의 글 미리보기 (②의 text 복사로 자동 — 카드 수정 없음)
- **옛 기록**: 사진별 글 데이터가 없으면 기존 단일 글 렌더 경로 그대로

## ④ 범위 제외 (YAGNI)

- 블로그·스트립·스냅 작성/보기 화면
- 옛 기록의 사진별 글 변환(마이그레이션)
- 소셜 피드 카드 구조 변경(미리보기 방식 유지)
- 사진별 글 검색·통계

## ⑤ 구현 시 준수 사항

- NewRecordScreen의 기존 state·저장 로직·모달(MediaPicker, 국가 검색, 날짜 선택 등)
  최대 재사용 — 리레이아웃이지 재작성이 아니다
- i18n: 새 문자열은 ko/en 키로 추가
- safe-area 컨벤션·디자인 토큰(#0A0A0F 계열, #BF85FC) 준수
- 커밋은 파일 단위 스테이징(작업 트리 WIP 주의)

## ⑥ 검증

- `npx tsc --noEmit` 통과
- 수동 시나리오:
  - 신규 작성 — 순서 뒤섞어 입력(글 먼저 → 국가 → 사진), 검증 스크롤·강조 동작
  - 사진 3장에 서로 다른 글 → 저장 → PostDetail에서 넘기며 글 전환 확인
  - 사진 삭제·순서 변경·대표 변경 시 글-사진 짝 유지
  - 대표 사진 글이 피드 목록 미리보기에 노출
  - 옛 기록 열람 — 기존 렌더 유지
  - 편집 모드 재저장, 여행 카드 경유 진입(기간 자동 적용)
  - 여행 기억 서랍(✨) 노출 유지
