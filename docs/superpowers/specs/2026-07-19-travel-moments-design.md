# 여행 기억(순간 메모) 기능 설계

날짜: 2026-07-19
상태: 사용자 승인 완료 (브레인스토밍 세션에서 단계별 확정)

## 개요

여행 중 순간적인 감정·기억을 즉시 한 줄로 적어두고, 나중에 정식 기록(피드·블로그·스트립)을
쓸 때 **참고 자료로 꺼내 보는** 기능. 새로운 기록 형식이 아니다 — 피드·통계·배지·소셜에
일절 섞이지 않는 개인 메모 레이어다.

핵심 철학: 순간 메모는 **기억을 되살리는 재료**다. 사용자가 서랍을 열어 "아, 이랬었지"를
떠올리고 자기 언어로 기록을 쓴다. 본문 삽입·복사 기능은 의도적으로 없다.

## 확정 결정 사항

| 항목 | 결정 |
|------|------|
| 캡처 내용 | 텍스트(필수) + 사진 1장(선택) |
| 감정 표현 | 무드 이모지 1개(선택) — 필수 아님 |
| 자동 컨텍스트 | 시간 + 위치(GPS→국가/지역). 권한 거부·실패 시 시간만 |
| 진입점 | **알림 단독** — 인앱 진입점 없음 |
| 알림 형태 | 해외 체류 중 상시 알림. 탭하면 캡처 시트 |
| 보관 | 프로필 여행 기록 카드의 "여행 기억" ✨ 아이콘 → 순간 목록 |
| 기록 작성 시 | 상단 배너 서랍, **순수 참고용**(탭하면 확대 보기만) |

## ① 데이터 모델

새 저장소 `momentStore` (기존 스토어 패턴 + `persist` 재사용):

```ts
interface TravelMoment {
  id: string;            // uuid
  text: string;          // 한 줄 메모 (필수)
  mood?: string;         // 이모지 1개 (선택)
  photoUri?: string;     // 로컬 사진 1장 (선택)
  countryName?: string;  // GPS 역지오코딩 결과 (실패 시 없음)
  regionName?: string;   // 가능할 때만
  createdAt: number;     // 자동 (epoch ms)
}
```

- `TravelRecord`와 완전 분리. 기록 수·방문국·배지 판정 등 어떤 집계에도 포함되지 않는다.
- 로컬 우선 저장. 서버 백업은 기존 appState 백업 채널(설정·기록 부가상태)에 텍스트/메타만
  편승한다. **사진은 1차 범위에서 로컬 전용** — 서버 업로드 없음.

## ② 알림 (유일한 진입점)

- 판정 시점: 앱 실행 및 포그라운드 전환 시.
  위치(권한 있으면 GPS, 없으면 최근 기록의 국가) → `homeCountryCode`와 비교.
- 해외로 판정되면 로컬 알림 게시:
  - **안드로이드**: `sticky`(ongoing) 고정 알림 — 스와이프로 지워지지 않음.
  - **iOS**: 상시 알림이 OS에 존재하지 않음 → 일반 로컬 알림. 사용자가 지웠어도
    다음 앱 실행/포그라운드 때 해외면 재게시(안드로이드에서 지워진 경우도 동일 규칙).
- 귀국 판정(위치가 거주국) 시 알림 자동 제거.
- 설정 > 알림에 "여행 중 순간 기록 알림" 토글 추가(기본 켜짐). 끄면 게시·재게시 모두 중단.
- 알림 탭 → 딥링크로 **캡처 시트**(모달) 오픈. 콜드 스타트(앱 종료 상태)에서도 알림
  응답으로 시트까지 도달해야 한다.
- 서버 푸시 불필요 — 전부 로컬 알림(expo-notifications). 오프라인에서도 동작.

### 캡처 시트

- 구성: 텍스트 입력 1칸 + 무드 이모지 행(😊🥹😮😌🤩 등, 단일 선택·해제 가능) +
  사진 첨부 버튼(1장) + 자동 위치 칩(예: "📍 오사카", 실패 시 미표시).
- 저장 → 토스트 "순간이 저장됐어요" → 시트 닫힘. 2초 안에 입력 시작이 목표.
- 오프라인 동작 필수: 역지오코딩 실패 시 좌표만 저장하지 않고 국가/지역 없이 저장한다.

## ③ 보관 — 여행 카드의 "여행 기억"

- 프로필(Travel archive) 여행 카드에 ✨ 아이콘 — 해당 여행에 귀속된 순간이 1개 이상일 때만 표시.
- 탭 → 그 여행의 순간 목록 시트: 시간순, 카드(무드 + 텍스트 + 사진 썸네일 + 장소·시각),
  길게 눌러 삭제(확인 후). 수정 기능 없음.
- **여행 매칭 규칙**: 순간의 `createdAt` + `countryName`이 여행 세션(출국~귀국, 국가 단위
  그룹핑 — 기존 규칙 그대로)의 국가·기간과 겹치면 그 여행에 귀속.
  `countryName`이 없는 순간은 기간만으로 매칭.
- 기록이 하나도 없는 여행의 순간: 카드가 없어 보관함으로는 접근 불가하지만 데이터는
  유지되며, 그 국가·기간으로 첫 기록을 쓸 때 작성 화면 서랍에서 노출된다.

## ④ 기록 작성 화면 서랍 (순수 참고용)

- 매칭되는 순간이 있을 때만 작성 화면 상단에 "✨ 이 여행의 순간 N개" 배너 표시.
- 펼치면 가로 스크롤 카드 서랍. 카드 탭 → 확대 보기 모달(전체 텍스트 + 사진 + 장소·시각).
- 삽입·복사 버튼 없음. 사용자가 보면서 직접 쓴다.
- 매칭: 작성 중인 기록의 국가 + 날짜 기준. 날짜 미입력 상태면 국가만으로 매칭.
  국가 정보가 없는 순간(역지오코딩 실패분)은 날짜가 겹칠 때만 노출.
- 피드·블로그·스트립 작성 화면 공통 적용.

## ⑤ 범위 제외 (YAGNI)

- 음성 메모
- 날씨 자동 기록
- 본문 삽입·인용 블록
- 무드 통계·회고 화면
- 순간 수정(삭제만 제공)
- 사진 서버 백업
- 인앱 캡처 진입점(FAB 길게 누르기 등) — 알림 단독 결정에 따름

## ⑥ 구현 시 준수 사항

- i18n: 새 문자열은 ko/en 키로 추가 (기존 react-i18next 체계).
- 안전영역: 새 시트/모달도 기존 safe-area 컨벤션을 따른다.
- 스토어: settingsStore/recordStore와 같은 패턴(Context + usePersistence).
- 위치→국가 변환은 기존 countryLocate 로직 재활용.
- 미커밋 WIP가 많은 작업 트리 특성상 커밋은 파일 단위 스테이징.

## ⑦ 검증

- `npx tsc --noEmit` 통과.
- 수동 시나리오: 캡처(온라인/오프라인, 사진 유/무, 무드 유/무) → 여행 카드 ✨ 목록 →
  작성 화면 배너/서랍 노출·확대 보기 → 삭제.
- 알림: 실기기에서 안드로이드(고정 알림·재게시·귀국 제거), iOS(재게시 규칙·콜드 스타트
  딥링크) 각 1회 이상 확인.
- 설정 토글 끔 → 알림 미게시 확인.
