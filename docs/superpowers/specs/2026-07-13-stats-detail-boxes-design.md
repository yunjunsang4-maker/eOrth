# 상세통계 화면 박스형 재구성 — 설계

날짜: 2026-07-13
대상 파일: `src/screens/StatsDetailScreen.tsx` (필요 시 `StatsScreen.tsx` 진입점 확인만)
시안: `iPhone 17 - 81.svg` (world travel 카테고리 예시)

## 배경 / 문제

현재 흐름은 **메인 통계 → 카테고리 탭 → 상세화면(히어로 1개 + 표 1개 + ‹›/› 셰브런으로 더 들어가는 단계)** 로, 상세 안에서 또 드릴다운하는 **이중 단계**가 있다.

시안처럼 각 카테고리 상세화면을 **한 페이지에 관련 상세통계를 여러 박스로 나눠** 보여주고, 내부 추가 드릴다운은 제거한다.

## 확정된 방향 (사용자 승인)

- **카테고리별 상세페이지 유지**: `world / yearly / region / countries / rating` 5개 진입점 그대로. 메인에서 탭한 항목의 상세로 진입하되, 그 안에서는 더 이상 드릴다운하지 않고 관련 박스만 세로로 나열.
- **히어로**: `world`만 지구본 시각 + ‹›로 대표 수치 순환. 나머지 4개는 **간단한 수치 히어로**(대표 숫자 + 라벨, 필요 시 작은 시각 1개, ‹› 순환 없음).
- **박스 스타일**: 시안의 마젠타→시안 그라데이션 테두리 박스. 기존 상세화면 표의 네온 스트로크 테두리 방식을 재사용.

## 화면 공통 구조

```
[헤더]  ‹  {카테고리 제목}
[히어로]  대표 수치 + 라벨 (world=지구본+‹›순환 / 나머지=수치)
[박스 1]  … 그라데이션 테두리 카드
[박스 2]  …
( [박스 3]  … )
```

- 진입/퇴장 애니메이션, 뒤로가기(‹)는 **기존 유지**.
- 카테고리 간 이동은 없음(뒤로가서 메인에서 다른 항목 탭). 상세 내부 드릴다운 셰브런 제거.

## 카테고리별 박스 정의

데이터 출처 표기: **[기존]** = 이미 계산 중, **[신규]** = 소량 신규 계산 필요(모두 `records`에서 파생 가능).

### ① world (세계·나라) — 시안 그대로
- **히어로(지구본 + ‹›순환)**: 대표 수치 순환 = 방문 나라수 → 방문 도시수 → 총 기록 → 총 여행일수 → 세계 커버리지%. **[기존]**
- **박스 `Recent Trips`**: 최근 여행 표 (열: 방문 나라 | 방문 도시 | 여행 기간 | 총 기록). 최근 4~5개, 종료일 기준 내림차순. **[신규]** 최근순 정렬·행 구성
- **박스 `가장 활발했던 해`**: `mostActiveYear` · `mostActiveCount`회 방문 + 첫 여행(`firstTravelYear` · `firstTravelLoc`). **[기존]**

### ② yearly (연도별)
- **히어로(수치)**: 총 방문 횟수(`totalYearlyVisits`). **[기존]**
- **박스 `연도별 방문 현황`**: 연도 | 방문 횟수 | 막대(비율). `yearlyItems`. **[기존]**
- **박스 `하이라이트`**: 가장 활발했던 해(`mostActiveYear`/`mostActiveCount`), 올해 방문 수, 활동 연 평균. **[신규]** 올해 수·연평균

### ③ region (대륙별)
- **히어로(수치)**: 방문한 대륙 수(`totalRegionCount`). **[기존]**
- **박스 `대륙별 방문 현황`**: 대륙 | 나라 수 | 비율. `regionItems`. **[기존]**
- **박스 `하이라이트`**: 최다 방문 대륙, 미방문 대륙 목록. **[신규]** 최다/미방문 산출

### ④ countries (국가별)
- **히어로(수치)**: Top 국가 + 방문 횟수(`countriesItems[0]`). **[기존]**
- **박스 `국가별 방문 횟수`**: 순위 | 나라 | 방문 횟수 (Top N). `countriesItems`. **[기존]**
- **박스 `하이라이트`**: 최근 방문 국가, 재방문 국가 수(방문 2회 이상). **[신규]** 최근/재방문 산출

### ⑤ rating (평점)
- **히어로(수치)**: 평균 별점(`avgRating`) + 별 표시. **[기존]**
- **박스 `별점 분포`**: 5★~1★ | 기록 수 | 비율. `ratingItems`. **[기존]**
- **박스 `하이라이트`**: 최고 평점 여행, 최근 평가, 평가한 기록 수(`ratedRecordsCount`). **[신규]** 최고/최근 평가 여행

## 컴포넌트 / 구현 방향

- `StatsDetailScreen`을 **박스 나열 렌더**로 전환:
  - `statType`별로 `hero`(수치/지구본 여부)와 `boxes: Box[]` 배열을 구성하는 데이터 셀렉터를 둔다.
  - `Box`는 `{ title, kind: 'table' | 'kv' | 'bars' , rows }` 형태의 작은 뷰 모델. 렌더러는 kind별로 분기.
  - **재사용 컴포넌트** `DetailBox`(그라데이션 테두리 카드) 하나로 모든 박스 렌더 → 파일 비대화 방지.
- 지구본 히어로는 기존 world 시각/‹› 순환 로직 재사용. 나머지는 수치 히어로(제목·숫자·라벨)로 단순화.
- 기존 단일 표/드릴다운 셰브런 로직 제거.

## 하지 않을 것 (YAGNI)

- 카테고리 통합 페이지, 상세 내부 드릴다운, 신규 지구본 시각(나머지 카테고리), 데이터 원본 스키마 변경.

## 성공 기준

- 5개 카테고리 상세화면이 각각 [히어로 + 관련 박스 2~3개]로 렌더되고, 내부 추가 네비게이션이 없다.
- world는 시안과 동일한 구성(지구본 + Recent Trips + 가장 활발했던 해).
- 모든 박스 데이터가 실제 `records` 기반으로 채워진다(더미 없음).
- `npx tsc --noEmit` 통과, 기존 화면과 충돌 없음.
