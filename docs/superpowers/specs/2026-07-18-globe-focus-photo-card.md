# 지구본 확대 시 초점 나라 고화질 사진 카드 설계

## 목표

사진 활성화 지구본을 **확대했을 때**, 화면 중앙에 오는(초점) 나라의 대표 사진을 **DOM 카드**로 띄워 작은 나라도 선명하게 보이게 한다. 전역 텍스처를 키우는 방식(메모리 벽)을 피하고, 원본 사진을 큰 카드로 렌더해 화질을 확보한다.

## 배경 / 제약

- 지구본은 전 세계를 하나의 등장방형 텍스처(현재 사진 모드 8192×4096)로 그린다. 작은 나라는 폴리곤이 몇 텍셀뿐이라 확대 시 흐릿하다.
- 텍스처를 더 키우면 16384≈512MB로 대부분 휴대폰이 크래시 → 불가.
- 이미 있는 것: `visitedMap[nameEn].photo`(대표 사진 URI, 풀해상도 원본이 `loadPhotoImage`로 캐시됨), `worldData.features`(국가 지오메트리 → `d3.geoCentroid`), 3D→화면 투영(`vec.clone().project(camera)`, ad-pin에서 사용), `#ad-layer` DOM 오버레이 패턴, 핀치 줌(`targetZ`/`currentZ` dolly).

## 설계 (전부 GlobeView.tsx의 3D 지구본 WebView 안)

### 트리거

- 조건: `globeDisplayMode === 'photo'` **그리고** 확대 상태(`currentZ`가 근접 임계값 이하 — 확대됨).
- 임계값·카드 크기는 상단 상수(`FOCUS_ZOOM_IN`, `FOCUS_CARD_PX`)로 조정 가능.

### 초점 나라 선택 (animate 루프, ~5fps로 스로틀)

1. 사진이 있는 방문국(visitedMap에 photo 존재)만 대상.
2. 각 나라 centroid(캐시: 최초 1회 `geoCentroid` 계산) → `geoToVec3` → 지구 회전 적용 → 카메라 정면(전면 반구, z>0) 여부 확인.
3. 전면인 나라 중 **화면 중심에 가장 가까운** 나라를 초점으로 선정. 없으면 카드 숨김.

### 카드 렌더

- `#focus-card`(overlay, `pointer-events:none`) — `<img>` + 얇은 흰 테두리·그림자. `object-fit:cover`, 크기 `FOCUS_CARD_PX`(예 120–140, 확대율에 살짝 비례).
- 위치: 초점 나라 화면 투영점 기준(살짝 위 오프셋). 프레임마다 갱신(회전/줌 따라 이동).
- src: `visitedMap[nameEn].photo`(원본) → 큰 카드여도 선명.
- 등장/퇴장: 확대 진입 시 페이드 인, 축소·초점 없음 시 페이드 아웃. 초점 나라가 바뀌면 크로스페이드(단순 src 교체 + 리셋).

### 상호작용

- 카드는 **비상호작용**(pointer-events none) — 지구본 드래그·줌 방해 없음. v1은 화질 프리뷰 전용(탭 액션 없음).

## 데이터 흐름

- RN 변경 없음. 이미 내려오는 `photo`/지오메트리만 사용. 카드 표시 여부·위치는 WebView 내부 계산.

## 파일 변경 목록

- `src/components/GlobeView.tsx` — 3D 지구본 HTML: `#focus-card` DOM + 스타일, animate 루프에 초점 선택·투영·카드 갱신 로직, 상수(임계값·크기), centroid 캐시.

## 검증

- `npx tsc --noEmit`.
- 수동: ① 사진 모드 확대 → 초점 나라 사진이 선명 카드로 뜸 ② 회전/줌에 카드가 따라 이동 ③ 축소 시 사라짐 ④ 작은 나라(싱가포르 등)에서 특히 개선 ⑤ 성능(프레임 드랍) 확인.

## 비목표 (YAGNI)
